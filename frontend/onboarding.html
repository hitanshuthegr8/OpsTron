Connect the OpsTron Agent

A lightweight sidecar that watches your containers and streams errors to OpsTron's AI engine in real-time. No inbound access, no code changes.

🐳 Docker Compose (Recommended)

Add this to your docker-compose.yml and label any container you want monitored:

opstron-agent: image: hitanshurathi/opstron-agent:latest restart: unless-stopped environment: OPSTRON_API_KEY: "your_api_key" volumes: - /var/run/docker.sock:/var/run/docker.sock:ro # Add this label to any service you want OpsTron to watch: your-app: image: your-app:latest labels: opstron.monitor: "true"

🖥️ Or run with a single Docker command

docker run -d \ -e OPSTRON_API_KEY="your_api_key" \ -v /var/run/docker.sock:/var/run/docker.sock:ro \ --name opstron-agent \ --restart unless-stopped \ opstron/agent:latest

Then label containers to opt them in: docker run --label opstron.monitor=true your-app
